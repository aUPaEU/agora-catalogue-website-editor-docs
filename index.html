Getting Started: Adding a Catalogue

  1. Enter Edit mode on the desired page.
  2. In the snippets panel, find the "Dynamic Snippet" under the "Custom Blocks" section.
  3. Drag and drop the snippet onto your page.
  4. Click on the newly added snippet to open the editor options on the right panel.

Main Configuration Options

The editor panel provides several options to customize your catalogue.

Data Source

This is the most crucial step. Here you define what information will be displayed.

  • Model: Select the Odoo model that contains the data you want to show (e.g., Products, Services, Events, Contacts). Once you select a model, other options will become available based on the fields of that model.
  • Sort by Field: Choose the field by which the items should be sorted (e.g., Name, Date, Price).
    Learn more
  • Filter Expression: (Optional) Add custom domain filters to limit the records displayed (e.g., only show products in stock, events in the future).
    Learn more

Visualization

Main View Type: Choose the main layout for your catalogue. Options: Grid, List, Calendar, Map.
See available layouts

Element Type: Select the style for each individual item in the list. Options: Default Card, Avatar Card, Collapsable Card, Adaptable Card.
See available cards

Fields

These options control what information appears on each card within the list.

  • Card Title: Select a field from your model to be used as the main title for each card.
  • Card Subtitle: Select a field to be used as a subtitle, appearing below the title.

Toggle

Visible Fields: Toggle visibility of fields. Relationship fields can be expanded. Important: These fields will only be visible in the detail view, not on the cards themselves.

Widgets

The web widgets, are custom HTML elements that could be selected to display an specific field of the model. Depending on the field type, the available widgets may vary. New widgets can be requested.

Available widgets
  • Tag: Renders a link to the related record for many-to-one or many-to-many relationship fields.
  • Url: Displays the field value within a styled badge. Ideal for status indicators or categorical data.
  • Url Chip: Displays the field value as an image. Ideal for fields containing image URLs or binary image data.

Sorting

Define the default order in which items will be displayed.

  • Order Criteria: Choose the field by which the items should be sorted (e.g., Name, Date, Price).
  • Order Type: asc or desc.

Filters

Add filters to allow users to narrow down results (e.g., Sales Team, Sales Person).
In example:

domain: [('field', '=', 'value')]

Learn more about domains

Additional Features

Enable or disable optional functionality for each element.

  • Enable Share Actions: Show social media sharing buttons on the detail view.
  • Enable PDF Reports: Allows users to download a PDF version of the item details. This requires a developer to first set up the report action. The Create Report Action option in the editor generates a printable PDF report (qweb-pdf) by creating an ir.actions.report in the backend. This is a developer-oriented feature.
  • Show Contact Button: Display a button to contact the responsible person.
  • Show Menu And Footer: Toggles the visibility of the main website navigation and footer, allowing the catalogue to be displayed in a more "embedded" or "full-page" mode.

Available Card Types

Default Card

This card will display a title, an image and a summary. This type is optimal for showcasing key information at a glance for elements where the image is important.

Avatar Card

Prominent circular image for people/contact lists.

Collapsable Card

Compact view with expandable details. This card is mostly recommended for elements with extensive info that should be accessible directly and doesn't have image. I.e. Funding opportunities.

Adaptable Card

Layout adjusts to different field types and lengths. I.e. If the card does not have image, the card will shrink. The layout will look in a Pinterest-like manner.

Detail View

When a user clicks an item they are taken to a detail page showing all relevant information. Configure Summary and Description fields in the editor, and any visible fields will appear as part of the detail layout.
